Java第7次课 继承与多态

发表于 2017-04-12 |

问题

  • 我们现在会写类，类中会写方法。
  • 写程序，要写多个类，假设这些类中有些功能是相同的。难道要把同一段代码一遍又一遍敲进去吗？就像同一句话，我们要反复说多遍。

继承

  • 继承的思想：把共同的程序代码放在某个类中，然后告诉其它类说这个类是它们的父类。
  • 继承关系意味着子类继承了父类的成员。（成员就是属性和方法）。
  • 现在有两个类，Child和Father。
  • 如果Child是Father类的子类，那么Child会自动地继承Father的属性和方法。
  • 但Child可以有自己的方法和属性。也可以覆盖掉继承自Father的方法。属性不能被覆盖掉，是因为不需要，它们并没有定义特殊行为。
  • 现在举个医生的例子，来看一下什么是继承。继承用extends关键字。

如何判断写的继承是否正确

  • 想要知道写的一个类继承另外一个类，继承的是否正确？
  • 用IS-A测试来检验。
  • 继承是单向的关系，外科医生是一个医生，医生可不是一个外科医生。

继承的意义

  • 通过提取出一组类间共同的抽象性，你能够排除掉重复的程序代码而将这个部分放在父类中。
  • 这样，如果有共同的部分需要改变，就只会有一个地方要修改而已，且这样的改变会应用到所有继承此行为的类。修改之后只需要重新编译就行，不必动子类。
  • 换上改变过的父类，则所有扩充过它的类都会自动使用到新的版本。
  • 定义出共同的协议。继承让你可以确保某个父型之下的所有类都会有父类所持有的全部方法。也就是说，你会通过继承来定义相关类间的共同协议。
