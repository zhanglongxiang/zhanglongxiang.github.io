怎么解决？

  • 需要一种方法来判断数组中元素是否已经被猜中，如果猜中了，下次再猜中就不能再算了。
  • 方法一：可以建立第二个数组，用第二个数组来标识第一个数组的状态。第一个数组存储坦克位置，第二个数组存储坦克每个部位是否被击中。
  • 缺点：太费时间，每次玩家猜中某一格时，都必须要改变第二个数组状态，而且还需要检查该数组来判断此格是否已经被命中。
  • 方法二：可以只用原来的数组，办法是将任何被击中的格子改为-1。这样只需要维护与检查一个数组。这种方案比方案一要好一些。
  • 缺点，就算有些格子已经被命中过了，还是得逐个搜寻数组中的那些格子。
  • 方法三：在命中某个格子时就把它删除掉，因此格子会越来越少。但是数组无法改变大小。因此必须作出新的数组并拷贝旧数组的值。这样仍然很麻烦。
  • 如果能找到一种数组会在删除掉某些元素时自动缩小就好了。这样就不必检查所有的元素，只要查询它是否带有寻找中的值就好。
  • 高兴的是，Java给我们提供了强大的工具类库，统称为Java API。其中在API中有一个类叫ArrayList，它能满足我们上面的要求。
  • ArrayList是什么，大家可以百度一下，网上很多详细介绍。
