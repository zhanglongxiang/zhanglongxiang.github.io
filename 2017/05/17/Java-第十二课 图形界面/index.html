Java第12次课 图形用户接口

发表于 2017-05-17 | 分类于 Java教学 |

先从窗口开始

  • 写程序总想写出一些图形界面，让程序看起来更象“软件”。这次课就来学习写图形界面。
  • 图形界面这部分，不需要我们自己写类，Java已经提供了很多图形工具类，大多数在javax.swing这个包下。
  • 本次课第一个图形工具类：JFrame，它代表程序的window，它就象一个容器，我们可以向里面加入其它的组件。

按动button怎么才能有反应？

  • 上面窗口的按钮只是个摆设，按下去没有反应。
  • 假设你要把按钮上面的文字在用户按下按钮时从”click me”变成”clicked”。如何让按钮被按下执行这个改变呢？
  • 用户按下按钮，就会产生事件。Java有许多不同的事件类型，大多数都与用户操作有关。
  • 如果想要知道按钮的事件，就会监听事件的接口。监听接口是介于监听（用户）与事件源（按钮）之间的桥梁。
  • 下面是事件的处理过程：
    1. 实现ActionListener这个接口;
    2. 向按钮注册（告诉它你要监听事件）;
    3. 定义事件处理的方法（实现接口上的方法）;

小总结：监听、事件源与程序

  • 作为程序员，要做的工作是实现接口并向按钮注册。
  • 作为事件源（按钮），工作是接受注册，取得用户事件，并在用户采取操作时调用监听的事件处理方法。
  • 作为事件对象（ActionEvent），作用是当作参数传进去，把事件的信息带给监听者。比如鼠标按下时，鼠标在屏幕上的坐标就是事件信息。

  • 当你的panel所处的frame显示的时候，paintComponent()这个方法就会被调用。
  • 你不用自己调用这个方法，它的参数是跟实际屏幕有关的Graphics对象，你无法取得这个对象，它必须由系统来交给你。
  • 你可以调用repaint()来要求系统重新绘制，然后系统会去调用paintComponent()这个方法进行绘制。
