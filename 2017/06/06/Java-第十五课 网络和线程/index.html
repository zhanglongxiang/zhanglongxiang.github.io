Java第15次课 网络和线程

发表于 2017-06-06 | 分类于 Java教学 |

序

  • Java程序可以通过网络连接其他计算机上的程序。
  • Java的一项好处是：只要网络畅通，网络程序其实就是进行远程数据流的读写操作（远程I/O）。

聊天程序概述

  • 包java.net提供了网络编程有关的开发工具类，在此包中有以下两个主要核心操作类：

    1. ServerSocket类，主要工作在服务器端，用于接收客户端请求。
    2. Socket类，每一个Socket对象都表示一个客户端。

  • ServerSocket类的常用方法：

    1. 构造方法，开辟一个指定的端口监听，一般使用5000以上的端口。

    2. 服务器端接收客户端请求，通过Socket返回。

    3. 关闭服务器端。

  • Socket类的常用方法：

    1. 构造方法，指定要连接的主机（IP地址）和端口。

    2. 取得指定客户端的输出对象。

    3. 从指定的客户端读取数据。

编写聊天客户程序

  • 我们用两阶段来编写聊天客户端应用程序。
  • 首先是只能发送信息给服务器，但不会收到其他来宾信息的版本。
  • 然后我们会发展出可接收和发送信息的完整版本。

可发送和接收

  • 重要问题：如何从服务器获得信息？
    1. 应该很容易，当你设定网络的同时也把输入串流建立好，然后使用readLine()读取信息。

  • 重要问题：何时会从服务器取得信息？

    1. 选择一：每隔20秒查询服务器一次
    2. 选择二：每当用户送出信息时就顺便从服务器读回信息
    3. 选择三：当信息被送到服务器上的时候就把它读回来

  • 我们选择第三种方法，我们需要程序同时执行的能力，检查服务器信息的同时不会打断用户与GUI的交互。

  • 这意味着我们需要新的线程，一个独立的执行空间。
  • 我们想要让只能发送的版本（第一版）可以维持原来的执行，同时有新的进程并行地读取服务器的信息并将它显示在文本区域上。
  • Java支持多线程，建立新的线程来执行是很简单的。
  • 就是这么简单，建立出新的线程对象就会启动新的线程，它是个独立的调用执行空间。

Java有多个线程但只有一种Thread类

  • Thread是java.lang这个包中的一个类。Thread的对象代表线程，当你需要启动新的线程时就建立Thread的实例。
  • 线程是独立的线程，它代表独立的执行空间。每个Java应用程序会启动一个主线程—将main()放在它自己执行空间的最开始处。Java虚拟机会负责主线程的启动。程序员负责启动自己建立的线程。
  • 要建立线程就得创建Thread对象。

有一个以上的执行空间代表什么

  • 当有一个以上的执行空间时，看起来会像是有好几件事情同时发生。
  • 实际上，只有真正的多处理器系统能够同时执行好几件事，但使用Java的线程可以让它看起来好象同时都在执行中。也就是说，执行动作可以在执行空间非常快速地来回交换，因此你会感觉到每项任务都在执行。
  • 顺序：
    1. Java虚拟机调用main()
    2. main()启动新的线程，新的线程启动期间main的线程会暂时停止执行。
    3. Java虚拟机会在线程与原来的主线程间切换直到两者都完成为止。
